
# Fix: Save Relative Indices for Groups to Preserve Superset/Circuit Groupings

## Problem Analysis

When a program is saved to Supabase:
1. Exercises get **new Supabase UUIDs** (generated by Postgres)
2. The `week_config.groups` array stores `exerciseIds` that reference the **frontend temporary IDs** (e.g., "exercise-123")
3. During assignment, the code tries to match these stale frontend IDs against the **new database UUIDs** → no matches → `groupId: null`

## Solution

Instead of saving the unstable `exerciseIds` array, save a **relative index array** (`exerciseIndices`) that references each exercise's 0-based position within that day's exercise list. These indices remain stable regardless of UUID changes.

### Example

If Day 0 has exercises `["Squat", "Bench", "Row"]` and a superset groups indices `[0, 1]` (Squat + Bench), the saved group looks like:

```json
{
  "id": "grp-123",
  "type": "superset",
  "exerciseIndices": [0, 1]
}
```

During assignment, we reconstruct the `groupId` by checking if the exercise's index exists in any group's `exerciseIndices`.

**Updated Code:**
```typescript
const weekConfig = weekPlan.map((day, i) => {
  const dayExercises = day.exercises;
  const rawGroups = dayGroups[i] || [];
  
  // Map each group to use relative indices instead of IDs
  const mappedGroups = rawGroups.map(g => ({
    id: g.id,
    type: g.type,
    exerciseIndices: g.exerciseIds
      .map(id => dayExercises.findIndex(e => e.id === id))
      .filter(idx => idx !== -1)
  }));

  return {
    label: day.label,
    notes: day.notes || "",
    blockType: day.blockType,
    groups: mappedGroups,
  };
});
```

**Updated Code:**
```typescript
const dayGroups = cfg?.groups || [];

// Sort exercises first to establish stable indices
const sortedExercises = (exs ?? []).sort((a, b) => (a.order_index ?? 0) - (b.order_index ?? 0));

const exercisesJson = sortedExercises.map((ex, exIdx) => {
  // Find if this exercise index belongs to any group
  const foundGroup = dayGroups.find(g => 
    g.exerciseIndices && g.exerciseIndices.includes(exIdx)
  );
  
  return {
    name: ex.name,
    sets: ex.sets ?? 3,
    reps: ex.reps ?? "10",
    rir: ex.rir ?? 2,
    failure_set: ex.failure_set ?? false,
    rest_time: ex.rest_time ?? "",
    notes: ex.notes ?? "",
    order_index: (ex.order_index ?? 0) % 100,
    groupId: foundGroup?.id ?? null,
  };
});
```

## Summary

- **Save:** Transform `exerciseIds` → `exerciseIndices` using `findIndex()` before saving to `week_config`
- **Load/Assign:** Match exercises by their sorted index position against `exerciseIndices` array
- **Benefit:** Superset/Circuit groupings survive database UUID regeneration

Single technical change, minimal footprint, solves the root cause.
